# /ideas — Cross-Domain Idea Generation

## Usage
```
/ideas
/ideas [focus-area]
```
Example: `/ideas` or `/ideas data-science projects`

## Purpose
Deep scan across all wiki domains to surface actionable ideas — project concepts, learning opportunities, tool ideas, and cross-domain connections that aren't obvious from individual pages.

## Steps

### 1. Gather Context
- Read `master-index.md` for all domain overviews
- Read `_meta/goals.md` for current priorities
- Read `_meta/profile.md` for current skill levels
- Read each domain's `index.md` for page catalogs

### 2. Scan for Patterns
Look for:
- **Knowledge intersections**: Where do two domains overlap? (e.g., data-science + marketing = marketing analytics)
- **Skill gaps as opportunities**: What's missing that would unlock new capabilities?
- **Project ideas**: What could you build combining skills from multiple domains?
- **Learning priorities**: Based on goals, what should you learn next?
- **Orphan knowledge**: Isolated pages that could connect to something valuable

### 3. Generate Ideas
Produce 5-10 actionable ideas organized as:
```
## Ideas Report — [date]

### Project Ideas
- [Idea]: [Why it matters] — Combines: [[domain1]], [[domain2]]

### Learning Priorities
- [Topic]: [Why now] — Unlocks: [capability]

### Cross-Domain Connections
- [Connection]: [[Page A]] + [[Page B]] → [insight]

### Quick Wins
- [Action]: [5-minute improvement to the wiki or workflow]
```

### 4. Offer Follow-Up
- "Want me to create a wiki page for any of these ideas?"
- "Want me to generate a `/roadmap` for one of these learning priorities?"
