# /connect — Cross-Domain Bridge Building

## Usage
```
/connect [domain1] [domain2]
/connect [topic1] [topic2]
```
Example: `/connect data-science marketing`

## Purpose
Find meaningful bridges between two domains or topics. Surface insights that emerge from combining knowledge across areas.

## Steps

### 1. Read Both Domains
- Read index files for both domains/topics
- Identify key pages in each

### 2. Find Bridges
Look for:
- **Shared concepts**: Same technique used differently (e.g., A/B testing in data science vs marketing)
- **Complementary skills**: One domain provides what the other needs
- **Transfer opportunities**: Patterns from one domain that apply to another
- **Project ideas**: Things you could build at the intersection

### 3. Present Bridges
```
## Connections: [Domain 1] <> [Domain 2]

### Bridge 1: [Title]
- From [[Page A]]: [concept]
- From [[Page B]]: [concept]
- Insight: [how they connect]

### Bridge 2: [Title]
...

### Potential Projects at the Intersection
- [Project idea combining both domains]
```

### 4. Create Links
- Offer to add `[[wikilinks]]` between connected pages
- Offer to create a new page documenting the connection
